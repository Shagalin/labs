## 1. Детектирование ключевых точек
Для начала подключаем библиотеки и загружаем 8 фотографий. 

Сглаживаем снимки фильтром Гаусса с ядром 5×5 и sigma=3 чтобы шум не испортил расчёты.

Далее с помощью оператора Собеля вычисляются производные по осям X и Y. Суть этого метода заключается в поиске градиента — показателя того, насколько быстро меняется яркость соседних пикселей. Чем резче переход от светлого к темному, тем выше значение градиента, что обычно соответствует границам объектов. Каждое ядро по очереди «проходится» по изображению и для каждого пикселя заново считает значение на основе его соседей. В результате получаем два отдельных изображения: одно содержит горизонтальные градиенты Gx, другое — вертикальные градиенты Gy

Градиент по X выделяет  места, в которых яркость заметно меняется слева направо или справа налево. Светлые участки на этом изображении соответствуют переходу от тёмного слева к светлому справа, тёмные — наоборот, от светлого слева к тёмному справа.

Градиент по Y отвечает за горизонтальные границы и реагирует на изменения яркости по вертикали. Светлые области означают, что яркость возрастает сверху вниз, тёмные — что она убывает сверху вниз (то есть возрастает снизу вверх).

Магнитуда градиента показывает «силу» края — насколько резким является переход между областями. Чем ярче пиксель на карте магнитуды, тем более выражен локальный контур.

Ориентация градиента кодирует направление края: оттенок цвета соответствует углу градиента, а яркость — его величине. 

Для анализа структуры изображения по каждому пикселю рассчитываются три компоненты: квадраты градиентов и их произведение. Из них строится структурный тензор (матрица вторых моментов), который кратко характеризует изменение градиента в окрестности пикселя и помогает определить тип локальной особенности (край, угол или однородная зона). Затем для каждого пикселя из тензора извлекаются собственные значения, а в карту отклика заносится меньшее из них. Принцип следующий:

оба значения малы — пиксель лежит в почти ровной области;

одно велико, другое мало — присутствует чёткий край в одном направлении;

оба значительны — градиент резко меняется в двух направлениях, что указывает на угол.

Поэтому минимальное собственное значение эффективно отражает «угловатость»: чем оно выше, тем вероятнее угол. Такой подход отсеивает простые края, выделяя зоны с сильными градиентными изменениями во всех направлениях.

Дальше идёт поиск локальных максимумов: изображение сканируется скользящим окном, пиксель признаётся углом лишь при превосходстве над соседями. Все кандидаты ранжируются по убыванию силы отклика, после чего отбираются лучшие.

# Вычисление дескрипторов (SIFT-метод)
SIFT (Scale-Invariant Feature Transform) — это алгоритм, который находит и описывает ключевые точки так, чтобы их можно было уверенно сопоставлять даже при изменении масштаба и поворота изображения.

Дескриптор — это числовой «отпечаток» ключевой точки: компактный вектор, который описывает её локальную структуру и позволяет отличать эту точку от других особых точек на изображении.

Вокруг каждой ключевой точки извлекается квадратный патч 16×16 пикселей, при этом область должна помещаться целиком в изображение, а точка — не быть у самого края.

Далее в патче для каждого пикселя определяются градиенты — их величина (магнитуда) и угол (ориентация), например, с помощью фильтра Собеля.

Затем патч делится на 16 ячеек 4×4 пикселя. Каждая ячейка превращается в гистограмму из 8 бинов (ориентаций): пиксель вносит вклад в бин по своему углу градиента с весом по магнитуде. Итог — 8 чисел на ячейку, или 128-мерный дескриптор для всего патча.

Все гистограммы склеиваются в единый вектор. Получается 16 ячеек × 8 бинов = 128 элементов.

Наконец, дескриптор нормализуется для устойчивости к освещению: компоненты приводятся к диапазону 0–1 с единичной нормой (сумма квадратов = 1). Затем каждое значение ограничивается сверху 0.2 — стандартный приём SIFT для снижения влияния резких градиентов.


## Сопоставление дескрипторов между кадрами

При сопоставлении между кадрами для каждого дескриптора с первого изображения ищется самый похожий дескриптор на втором. Похожесть измеряется евклидовым расстоянием в 128‑мерном пространстве: чем оно меньше, тем более похожи точки. Для надёжности берут не одного, а два ближайших соседа и сравнивают их расстояния d1 и d2.

Если лучшее совпадение заметно выигрывает у второго (условие d1<0.75⋅d2), пара считается надёжной. Такой порог отсеивает неоднозначные совпадения, когда дескриптор одинаково хорошо «подходит» сразу к нескольким точкам.

## Построение траектории
RANSAC (RANdom SAmple Consensus) — итеративный метод оценки параметров модели, который остаётся надёжным даже при наличии выбросов в данных. Вместо того чтобы подгонять модель по всем точкам сразу, он многократно строит её по небольшим случайным подмножествам и выбирает решение, которое лучше всего объясняет большинство наблюдений без выбросов.

Проблема в том, что заранее неизвестно, какие точки являются выбросами. Поэтому на каждой итерации берётся минимальное число соответствий, достаточное для задания модели, по ним оцениваются параметры, а затем для всех остальных точек проверяется, насколько хорошо эта модель их описывает.

В настройках используется 100 итераций — этого обычно достаточно, чтобы с высокой вероятностью найти корректную гипотезу. Порог ошибки принят равным 10 пикселям: точка считается согласующейся с моделью (inlier), если расстояние между её предсказанным и реальным положением меньше этого значения.
