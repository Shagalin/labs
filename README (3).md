# Визуальная одометрия (навигация)
## Цель: Разработать систему визуальной одометрии (навигации) по группе фотографий
## 1. Детектирование ключевых точек
Работа начинается с загрузки 8 изображений. Чтобы мелкий шум не исказил дальнейшие расчеты, снимки предварительно сглаживаются фильтром Гаусса с ядром 5×5 и sigma=3.

Затем с помощью оператора Собеля вычисляются производные по осям X и Y. Суть этого метода заключается в поиске градиента — показателя того, насколько быстро меняется яркость соседних пикселей. Чем резче переход от светлого к темному, тем выше значение градиента, что обычно соответствует границам объектов.

Математически это реализуется через свёртку: по изображению «проходятся» два специальных фильтра (ядра), реагирующие на изменения яркости по вертикали и горизонтали. В результате алгоритм находит как силу перепада яркости, так и его направление.

Каждое ядро по очереди «проходится» по изображению и для каждого пикселя заново считает значение на основе его соседей. В результате получаем два отдельных изображения: одно содержит горизонтальные градиенты Gx, другое — вертикальные градиенты Gy

Градиент по X выделяет вертикальные границы, то есть места, где яркость заметно меняется слева направо или справа налево. Светлые участки на этом изображении соответствуют переходу от тёмного слева к светлому справа, тёмные — наоборот, от светлого слева к тёмному справа.

Градиент по Y отвечает за горизонтальные границы и реагирует на изменения яркости по вертикали. Светлые области означают, что яркость возрастает сверху вниз, тёмные — что она убывает сверху вниз (то есть возрастает снизу вверх).

Магнитуда градиента показывает «силу» края — насколько резким является переход между областями. Чем ярче пиксель на карте магнитуды, тем более выражен локальный контур.

Ориентация градиента кодирует направление края: оттенок цвета соответствует углу градиента, а яркость — его величине. Например, синий может соответствовать углу 0°, фиолетовый — 90°, пурпурный — 180°, оранжевый — 270°.

Для анализа структуры изображения далее для каждого пикселя вычисляются три величины: dx^2, dy^2 и dx⋅dy. На основе этих значений формируется структурный тензор (матрица вторых моментов) — матрица, которая компактно описывает, как меняется градиент в окрестности точки и позволяет судить о наличии и типе локальных особенностей (край, угол, однородная область). На следующем этапе из структурного тензора для каждого пикселя вычисляются собственные значения λ1 и λ2. В карту отклика записывается минимальное из них. Логика такая:

если оба собственных значения малы — это почти однородная (плоская) область;

если одно большое, а второе маленькое — в этой точке есть выраженный край в одном направлении;

если оба значения велики — яркость сильно меняется сразу в двух направлениях, то есть с высокой вероятностью это угол.

Поэтому минимальное собственное значение хорошо характеризует «угловатость» точки: чем оно больше, тем точка больше похожа на угол. Такой критерий позволяет отфильтровать простые края и оставить только участки с сильными изменениями градиента во всех направлениях.

Далее выполняется поиск локальных максимумов: изображение просматривается скользящим окном, и пиксель считается углом, только если его значение максимальное в своей окрестности. После этого все найденные углы сортируются по убыванию «силы» (значения меры угловатости), и из них выбирается заданное число лучших точек, например 25.

# Вычисление дескрипторов (SIFT-метод)
SIFT (Scale-Invariant Feature Transform) — это алгоритм, который находит и описывает ключевые точки так, чтобы их можно было уверенно сопоставлять даже при изменении масштаба и поворота изображения.

Дескриптор — это числовой «отпечаток» ключевой точки: компактный вектор, который описывает её локальную структуру и позволяет отличать эту точку от других особых точек на изображении.

Сначала вокруг каждой найденной ключевой точки вырезается квадратный патч размером 16×16 пикселей. При этом проверяется, чтобы эта область полностью находилась внутри изображения и ключевая точка не была слишком близко к границе.

Затем внутри патча для каждого пикселя вычисляются градиенты — их магнитуда (сила) и ориентация (направление), например с помощью оператора Собеля, как было описано выше.

После этого область 16×16 разбивается на 16 ячеек по 4×4 пикселя. В каждой ячейке строится гистограмма ориентаций с 8 бинами (направлениями): вклад каждого пикселя добавляется в тот бин, который соответствует его углу градиента, с весом, равным магнитуде. В итоге для каждой ячейки получается 8 чисел, а для всего патча — набор из локальных гистограмм, образующий дескриптор.

[<img width="1170" height="519" alt="image" src="https://github.com/user-attachments/assets/cde42b60-999a-43bf-998f-21a8df51a9ad" />]

В конце все локальные гистограммы объединяются в один вектор. Шестнадцать ячеек по 8 значений дают дескриптор длиной 128 элементов (16 × 8).

Далее дескриптор нормализуется, чтобы сделать его менее чувствительным к изменениям общего освещения: значения масштабируются так, чтобы лежать в диапазоне 0–1, а сумма квадратов компонент была равна 1. После этого каждое значение обрезается сверху порогом 0.2 — классический приём в SIFT, который уменьшает влияние слишком сильных градиентов и делает дескриптор более устойчивым.

## Сопоставление дескрипторов между кадрами
При сопоставлении между кадрами для каждого дескриптора с первого изображения ищется самый похожий дескриптор на втором. Похожесть измеряется евклидовым расстоянием в 128‑мерном пространстве: чем оно меньше, тем более похожи точки. Для надёжности берут не одного, а два ближайших соседа и сравнивают их расстояния d1 и d2.

Если лучшее совпадение заметно выигрывает у второго (условие d1<0.75⋅d2), пара считается надёжной. Такой порог отсеивает неоднозначные совпадения, когда дескриптор одинаково хорошо «подходит» сразу к нескольким точкам.

## Построение траектории
RANSAC (RANdom SAmple Consensus) — итеративный метод оценки параметров модели, который остаётся надёжным даже при наличии выбросов в данных. Вместо того чтобы подгонять модель по всем точкам сразу, он многократно строит её по небольшим случайным подмножествам и выбирает решение, которое лучше всего объясняет большинство наблюдений без выбросов.

Проблема в том, что заранее неизвестно, какие точки являются выбросами. Поэтому на каждой итерации берётся минимальное число соответствий, достаточное для задания модели, по ним оцениваются параметры, а затем для всех остальных точек проверяется, насколько хорошо эта модель их описывает.

В настройках используется 100 итераций — этого обычно достаточно, чтобы с высокой вероятностью найти корректную гипотезу. Порог ошибки принят равным 10 пикселям: точка считается согласующейся с моделью (inlier), если расстояние между её предсказанным и реальным положением меньше этого значения.
